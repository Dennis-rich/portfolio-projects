## Цель проекта
Разработка модели машинного обучения для прогнозирования оттока клиентов телекоммуникационной компании с приоритетом на метрику **Recall**, чтобы максимизировать обнаружение уходящих клиентов при сохранении приемлемой точности.

---

## Ключевые результаты

### Модель
- **Алгоритм:** Логистическая регрессия (L1-регуляризация)  
- **Основная бизнес-метрика:** F2-score  
- **Оптимизация гиперпараметров:** Optuna (40 trials)

### Метрики на тестовой выборке:
| Метрика | Значение |
|--------|----------|
| Precision | 0.55 |
| Recall | 0.74 |
| F1-score | 0.63 |
| F2-score | 0.70 |

### Оптимальные параметры:
- **C:** 0.093  
- **Class weight:** `{0: 1, 1: 4}`  
- **Penalty:** L1  
- **Порог классификации:** 0.6583  

---

## Бизнес-контекст
Телеком-компании работают на высококонкурентном рынке, где стоимость привлечения нового клиента значительно выше стоимости удержания текущего. Отток приводит к:
- снижению выручки,
- росту затрат на маркетинг,
- падению LTV и других ключевых метрик.

**Бизнес-цель:** выявлять клиентов с повышенной вероятностью оттока и применять стратегии удержания заранее.

---

## Решение

### Выбор метрик
Главная проблема — **False Negative**, когда реальный уходящий клиент не был отмечен моделью. Поэтому приоритет метрик:

1. **Recall** — максимальная полнота  
2. **F2-score** — контроль качества с акцентом на Recall  
3. **Precision** — минимизация ложных срабатываний  

### Анализ важности признаков

#### Факторы, увеличивающие отток:
1. Высокие ежемесячные платежи (+0.87)  
2. Электронные чеки (+0.37)  
3. Безбумажный биллинг (+0.32)  
4. Оптоволоконный интернет (+0.18)

#### Факторы, уменьшающие отток:
1. Долгосрочные контракты (–0.86)  
2. Наличие телефонной услуги (–0.86)  
3. Онлайн-безопасность (–0.47)  
4. Техническая поддержка (–0.37)

## Методология

### Этапы работы:
1. **Предобработка данных**
   - Обработка пропусков (TotalCharges)
   - Приведение типов
   - Кодирование категориальных признаков
   - Feature engineering

2. **EDA**
   - Анализ баланса классов
   - Корреляционный анализ
   - Анализ категориальных и числовых признаков

3. **Разработка модели**
   - Optuna (40 итераций)
   - Стратифицированная кросс-валидация (5-fold)
   - Learning curves
   - Анализ bias–variance

4. **Тестирование**
   - Hold-out выборка
   - Ручные примеры
   - Проверка стабильности модели

### Сильные стороны подхода
- Оптимизация под бизнес-метрику  
- Интерпретируемость модели  
- Комплексный анализ данных  
- Стойкость на новых данных

## Бизнес-рекомендации

### Что внедрить в первую очередь:
1. Продвигать долгосрочные контракты  
2. Включать телефонную услугу в базовые тарифы  
3. Предлагать онлайн-безопасность как стандарт  
4. Пересмотреть тарифную политику (высокие платежи → основной драйвер оттока)

### На что обратить внимание:
1. Клиенты с электронными чеками — зона риска  
2. Новые клиенты с высокими платежами — критические первые 3–6 месяцев  
3. Пользователи оптоволоконного интернета — повышенный риск  
4. Клиенты на помесячных тарифах — главная источник оттока

### Готовность к внедрению
